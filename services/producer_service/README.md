# Producer Service

## Overview

Simulates a real-time sensor feed by reading a pre-generated Parquet dataset row by row and publishing each record to the Redpanda `telemetry-data` topic. It is the **entry point of the online pipeline** — nothing in the streaming stack processes data until this service runs.

## File Structure

```
services/producer_service/
├── dockerfile
├── config/
│   └── config.py       # Kafka settings, dataset path, batch parameters
└── src/
    └── producer.py     # Reads Parquet → serialises → publishes to Redpanda
```

## How It Works

```
data/synthetic_datasets/industrial_washer_with_anomalies_streaming/
  (Parquet — generated by create_datasets_service)
        │
        ▼
  pandas.read_parquet()
        │
        ▼
  for each row:
    serialize(key=Machine_ID, value=row.to_dict())
        │
        ▼
  producer.produce()   → Redpanda  →  telemetry-data topic
        │
  every 3 messages: sleep(1s)      ← real-time simulation throttle
```

Messages are keyed by `Machine_ID`, preserving per-machine ordering in Redpanda.

## Batch / Throttle Parameters

| Parameter | Value | Effect |
|---|---|---|
| `BATCH_SIZE` | `3` | Produce 3 messages, then pause |
| `BATCH_DELAY_SECONDS` | `1` | Pause duration between batches |

With 3 machines reporting in sync every second (matching the dataset generation cadence), one batch = one timestamp across all machines → 1 simulated second per real second.

## Configuration (`config.py`)

| Setting | Default | Description |
|---|---|---|
| `KAFKA_SERVER` | `redpanda:9092` | Redpanda broker address |
| `TOPIC_TELEMETRY` | `telemetry-data` | Target topic |
| `DATA_DIR` | `/producer/data/streaming_data` | Bind-mounted data directory |
| `STREAMING_DATASET` | `industrial_washer_with_anomalies_streaming` | Parquet folder name inside `DATA_DIR` |
| `BATCH_SIZE` | `3` | Messages per batch before throttle |
| `BATCH_DELAY_SECONDS` | `1` | Sleep between batches (seconds) |

`STREAMING_DATASET` can be overridden at runtime via the `STREAMING_DATASET` environment variable to point at a different dataset (e.g. the normal-only variant).

## Expected Input

The producer reads from:
```
data/synthetic_datasets/industrial_washer_with_anomalies_streaming/
```

This directory is produced by `create_datasets_service` (`save_datasets(..., streaming=True)`). It must exist and be non-empty before this service starts.

## Build & Run

```bash
# Build
docker build -f services/producer_service/dockerfile -t producer_service:latest .

# Run
docker compose run --rm producer_service
```

The service exits automatically once all rows in the dataset have been published (`restart: "no"` in compose).
